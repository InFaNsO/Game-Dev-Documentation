# BGamer — Content Formula: The Game Breakdown

> [!important] This is the channel's content format **going forward — as of 2026-06-20.**
> It stays the **default** until we gather more data/information that tells us to revise it. Treat this doc as the source of truth for how long-form videos and shorts are made.

## The formula (one line)
Take a big, popular, on-niche game (the **anchor**), break it down across its strongest aspects, and for each aspect **recommend + mini-review** other games that nail that same aspect. The anchor brings the audience (reach); the recommendations are the payload (our identity + the reason to subscribe).

## What we break a game down by — the aspects
**Primarily: Gameplay (mechanics) · Theme · Narration / story.**
But it can be **any** aspect that has a deep bench of comparable games — explicitly including **Soundtrack** and **Art style**, plus setting / atmosphere, characters & relationships, and emotional beats.

**Rule:** for each anchor, pick the aspect (or a mix of 2–3) that (a) the game is most distinctive for, and (b) maps to the deepest set of *recognizable, on-niche* games.
- Mechanic-rich game → break down by **gameplay** (e.g. Expedition 33: parry, tells, builds).
- Mechanically unique but thematically rich game → break down by **theme / narration** (e.g. Pragmata: the Hugh & Diana bond, lonely sci-fi, AI).
- A **theme** axis often pulls *more* demand than mechanics, because demand lives in the theme ("games like The Last of Us").

## Long-form structure (the reusable skeleton)
- **0:00–0:15 — Hook:** big claim + what's coming + tease the best pick. ("…and the last one might do it better than [anchor].")
- **0:15–0:40 — Frame:** one sentence on the anchor, then set up the N aspects (use YouTube chapters).
- **Body — one repeatable unit per aspect:** explain the aspect *in the anchor* (30–45s b-roll) → then 1–2 **30–50s mini-reviews** of games that nail that aspect, focused only on that aspect.
	- **Build each mini-review as a self-contained block** (names the game + aspect + why; clean in / out points) so it can be lifted straight into a short.
- **Close:** deliver the teased pick → engagement question ("which aspect's your favourite — what'd I miss?") → "subscribe for more breakdowns" → end screen to the strongest mini-review.

Pick the strongest **4–5 aspects** → ~8–12 min video → **8–10 shorts**.

## Shorts — "Bookend Loop" (reuse the long-form clips)
We **reuse the already-edited mini-review clips** from the long-form. Each short = small **intro → the reused clip → outro**, engineered to **loop seamlessly** (strict requirement).

**Template:**
```
[anchor frame ~0.5s + HOOK]  →  [reused long-form clip, 20–45s]  →  [anchor frame ~0.5s + CTA]  ↺
```
- **Intro (1.5–3s):** open on a signature "anchor frame" of the game + hook text + one VO line. Lands in <1s.
- **Body (20–45s):** the exact mini-review segment lifted from the long-form. **No re-editing.**
- **Outro (1.5–3s):** return to the **same anchor frame** → invisible visual seam. CTA as a **text overlay on that frame**, never a separate silent card.

The anchor-frame bookend is the key: intro and outro carry different *text* while the *image* loops perfectly.

### Seamless loop — strict requirement (the 3 seams)
1. **Visual seam:** first frame ≈ last frame (the anchor-frame bookend gives this for free).
2. **Audio seam:** one low music bed under the whole short, on a track that loops cleanly (end bar → start bar). Never swell to a hard stop.
3. **Narration seam:** write VO so the last line flows into the first — end *"…hits harder than [anchor] —"* ↺ start *"— if you loved that in [anchor], play this."* The narration never "ends."

### Loop checklist (every short must pass)
- [ ] First frame ≈ last frame
- [ ] Music continuous, no hard ending
- [ ] Last VO line leads into first VO line
- [ ] No silent end card / no fade-to-black
- [ ] CTA overlaid on footage, not a trailing card
- [ ] Loops 3× with no findable seam

### Reuse pipeline (edit once → ship 8–10)
- Build each long-form mini-review as a **self-contained block** (stands alone out of context).
- **Frame for vertical:** long-form is 16:9, shorts are 9:16 — keep action **center-frame** so it survives the crop, and keep text in the vertical safe zone. *(Biggest practical gotcha.)*
- A short = the lifted block dropped between the two bookend templates → export 9:16. Only the hook text, anchor frame, and CTA swap.
- Drip 1 short every 1–2 days for ~2 weeks after the long-form; each ends "full breakdown on the channel."

## Why this fixes our numbers (rationale)
*(From the 2026-03 → 06 analytics diagnosis.)*
- **CTR:** recognizable anchor in title + thumbnail (show the game + a 2-word promise; test 2–3 variants). Recent long-forms got 9k–33k impressions but died at **0.3–1.8% CTR** — packaging was the bottleneck, not demand.
- **Retention:** segmented by aspect (each segment re-hooks), the 30–50s discipline forces pacing, the cold-open tease pulls to the end. Fixes our **13–28% avg-viewed** bleed.
- **Niche coherence:** every episode is "deep single-player breakdown + recommendations," so an audience finally compounds. We were at **+1 net sub / 7,613 views**.
- **Bonus:** breaking down design quietly marks us as people who understand games — the no-devlog bridge to selling our own games.

## Measurement rule
Commit to this format for **~6 videos** before judging. Vary the anchor game, **never the format**. Judge by the **median over 6**, not any single upload. Targets: **CTR ≥ 4%**, **avg % viewed ≥ 35%**. The shorts also give per-aspect performance data → use it to pick future anchors.
