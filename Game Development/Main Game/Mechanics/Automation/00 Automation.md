# Automation Game — Hub

> The **Automation pillar** testbed: a **spell-gear workshop** game set in the pre-Breach **Mani Accord golden age**. The player **designs spells** (the signature *Conjuration Engineering* mechanic), **mass-produces** them through an automation workshop, and **sells** them in a **competitive market** against rival AI workshops.

## One-line pitch

Run a Mani spell-gear workshop in the Accord's frontier boom: **engineer** spells as conjured 3D elemental constructs, **automate** their production, and **out-compete** rival workshops in a living market.

## Design DNA (the three references)

- **Modulus** → the core verb: *design the process, don't pick a recipe.* You build a process that **produces** a 3D construct. (Plus its cautionary lesson — it lacked tech-tree depth; we fix that with an operations tech-tree.)
- **Dyson Sphere Program** → the payoff: **creation made visible** (watch the spell get produced) + scale.
- **The Good Company** → the pressure: a **living market + AI rival** + design-product-by-components. Its anti-lesson is a hard rule: **no worker/machine/hybrid split** — one production system.

## The two automation surfaces

- **Surface A — Spell Design ("Conjuration Engineering").** The deep, signature layer. Build a modular process on the gem grid that **produces and launches** a 3D elemental construct (Form + Substance + Behavior). The form comes from *shapers* (process-operations), never hand-placement. See `06`.
- **Surface B — Workshop Production.** The familiar "regular automation" layer (belts/machines, **fully machine-automated**) that **manufactures the parts + gear** Surface A needs and feeds a **mega assembler** that visibly builds each finished product. **Its throughput = how much you can put on the market.** Coupled to A by a shared part vocabulary. See `05`.
