# Automation — Scope & Depth Allocation

> The DEEP / BOUNDED / CUT ledger — same discipline as [[08 Scope & Depth Allocation|the city builder's]]: depth concentrated on what the dream game runs on; everything else leashed in advance or consciously cut with the reason recorded. **LOCKED 2026-06-10.**

---

## DEEP (what this game exists to prove)

- **Surface A — Conjuration Engineering**: full module tech tree, max-depth mixing (all 14 elements), mold/SDF constructs, the derived-stat model incl. cast time. *(The signature — `06`, `09`.)*
- **Surface B — workshop production**: parts model, difficulty-tiered chains, the mega assembler, space-constrained routing. *(`05`, `09`.)*
- **The A↔B coupling** — modules = parts, blueprint = BOM. *(The keystone, `09`.)*
- **Market + competitor AI v1** — the cross-portfolio AI thread's debut. *(`07`.)*
- **Progression spine** — analysis machines, Money + RP, reputation/frontier. *(`08`.)*

## BOUNDED (needed, but with a pre-drawn ceiling)

- **Part catalog ~15–25** · **element raws ~4** · **machine types ~6** · **gear 4 types × 3 slots** · **~8 render templates**
- **One warehouse room** + expansion steps (no second building, no multi-site)
- **Competitor: 2–4 rivals**, utility/heuristic, behind-the-curtain (v1 leash, `07`)
- **Customers = market screen + named contract clients + ambient walk-ins.** Selling and all its strategy happen on the market board and in written commissions. Customers DO appear in the shop — but as **set dressing only** (wander in, admire, leave; zero interaction). One free touch: their **frequency is driven by your actual sales rate**, so a successful shop visibly bustles — ambience reflects the sim without being a system.
- **Story: light**, delivered via bake-off contracts/correspondence (the plot session will hang on this channel)

## CUT (consciously not built — with reasons)

- **Combat** — you never fight; the **Test Room is the only firing range**. Spell *data* feeds the combat pillar, but this game has no enemies, health, or fights. (Combat = Games 1–2's pillar.)
- **Interactive retail / shop-layout layer** — customers you serve, shelf placement affecting sales. **Failed the portfolio reuse test (user's rule, 2026-06-10):** no other game in the portfolio uses agent-retail or interior-layout-driven sales (the city pillar explicitly went radius-based, not agent-based), so the system transfers nowhere. **→ Production-time note: revisit as a flavor layer if time & budget allow.** The cosmetic slice that *does* reuse survives as the ambient walk-ins above.
- **In-house gem refining** — gems bought pre-refined (locked `05`; refining minigames = the Looter Shooter's identity).
- **Worker/staff sim** — fully machine-automated (locked `05`).
- **Multi-site logistics** — one warehouse; shipping networks are the city pillar's job.
- **Embodied competitor (v2)** — lives in the city fusion (`07` thread plan).
- **Procgen** — the **warehouse (Surface B) is hand-authored**, and contracts/market events are systemically generated. **EXCEPTION (added 2026-06-10): the gem grid (Surface A) IS spatially procgen** — each gem has a generated internal layout with holes + obstructions (see `06`). So automation is no longer "no procgen": gem-grid topology is in; full level / dungeon / island procgen stays out.
- **Mobile** — tech note: VFX Graph doesn't support mobile → desktop only (we weren't targeting mobile anyway).
